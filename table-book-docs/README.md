## Что за проект

**TableBook** — SaaS для бронирования столиков в ресторанах. Пишем на Spring Boot. Учусь Java/Spring через реальный проект.

## Workflow

Я (Batyrkhan) и Claude работаем в **Claude Desktop с MCP filesystem**. Длинный диалог = много токенов, поэтому модель такая:

1. **В рамках одной фазы** → один чат
2. **В конце фазы** → Claude пишет всю документацию в этот волт (`TableBook/`)
3. **Старый чат удаляется**, токены освобождаются
4. **Новый чат** начинается со слов "прочитай файлы в волте TableBook/ и поехали с Фазы N"
5. Claude читает файлы → понимает контекст → продолжает работу

Это намеренное ограничение для эффективного использования токенов.

## Иерархия папок

Документация разделена на три папки по характеру содержимого:

```
TableBook/
├── README.md          ← этот файл, точка входа
├── static/            ← статичные / редко меняющиеся данные
├── phases/            ← фазы создания проекта (планы и архивы)
└── progress/          ← обновляющиеся файлы с текущим прогрессом
```

### `static/` — статика

Файлы, которые **практически не меняются** от чата к чату. Читаются в начале каждого нового чата.

| Файл | Что | Как часто меняется |
|------|-----|---------------------|
| `static/profile.md` | Кто я (Batyrkhan), мой стиль, чего НЕ делать | редко (по мере накопления анти-уроков) |
| `static/architecture.md` | Архитектурные решения проекта (LOCKED) | очень редко |
| `static/spring-cheatsheet.md` | Шпаргалка по Spring | пополняется по мере изучения |

### `phases/` — фазы создания проекта

План текущей фазы и архивы завершённых. Архивные файлы помечаются суффиксом `-done`.

| Файл | Что |
|------|-----|
| `phases/phase-N-plan.md` | План текущей или ближайшей фазы |
| `phases/phase-N-<topic>-done.md` | Архив завершённой фазы: что делали, какие решения, ошибки |

`-done` файлы — это **архив**. Не нужно читать в начале каждого чата, **но обращаться при необходимости** (например, "почему мы так решили в Фазе 1?" → читать `phases/phase-1-auth-done.md`).

### `progress/` — текущий прогресс

Файлы, которые **обновляются каждую фазу** и показывают актуальное состояние проекта.

| Файл | Что | Как часто меняется |
|------|-----|---------------------|
| `progress/progress-report.md` | Где мы сейчас, тестовые данные, часы, что отложили | каждую фазу |

## После каждой фазы — что обновлять

В конце каждой фазы Claude должен:

1. **Обновить `progress/progress-report.md`** — добавить новые часы, обновить прогресс-бары, новые уроки
2. **Создать `phases/phase-N-<topic>-done.md`** — детальный архив только что закрытой фазы
3. **Создать `phases/phase-N+1-plan.md`** — план следующей фазы (если ещё нет)
4. **Удалить** старый `phases/phase-N-plan.md`, если он стал `-done`
5. (Если изменились архитектурные решения) — обновить `static/architecture.md`
6. (Если выучили новые темы) — дополнить `static/spring-cheatsheet.md`
7. (Если появились новые анти-уроки про Claude) — дополнить `static/profile.md`
